## Нефункциональные требования

<!--
По атрибутам качества (см. ISO/IEC 25010 [ссылка]).

### Производительность
- Время отклика API (p95) ≤ 200 мс на лёгких эндпоинтах.
- Время инференса 3D-модели ≤ 30 с.
- Throughput одного ML-воркера на GPU: ≥ 30 моделей/мин (целевая).

### Масштабируемость
- Горизонтальное масштабирование ML-воркеров (добавление инстансов без даунтайма).
- Stateless design API-слоя.

### Надёжность и отказоустойчивость
- При падении ML-воркера — задача не теряется, переоформляется на другого воркера.
- БД с регулярным бэкапом.
- Graceful degradation при недоступности компонента.

### Доступность
- Целевая ≥ 99% (типичный SLA для бакалаврской работы).
- Мониторинг и алерты.

### Удобство разработки и эксплуатации
- Контейнеризация всех компонентов.
- Структурированные логи в едином формате.
- Метрики для дашбордов.

### Ограничения по железу
- Один-два потребительских GPU (например, NVIDIA RTX серии).
- Не более 16 ГБ VRAM на воркер.
- CPU-инференс как fallback (с увеличенным временем).

Каждый пункт — со ссылкой на источник (книги по архитектуре) или внутреннее обоснование.
-->
