## 5.1 Постановка задачи дообучения

<!--
Объём: 1.5-2 страницы.

## 5.4 Процедура дообучения

<!--
Объём: 2-2.5 страницы.

### 5.4.1 Гиперпараметры
- Learning rate: 1e-5 (низкий, для сохранения претрейна).
- Optimizer: AdamW (β1=0.9, β2=0.999, weight decay=1e-4).
- Scheduler: cosine annealing.
- Epochs: 5-10.
- Batch size: 1-4 (ограничение по VRAM).

### 5.4.2 Стратегия заморозки
- Заморозка vision transformer encoder.
- Обучение последних 2-3 блоков triplane decoder.
- Обоснование: основные visual features — общие, специфика домена — в декодере.

### 5.4.3 Loss функция
- Композитный loss: rendering loss + chamfer loss + perceptual loss.
- Веса компонент.

Формула 5.1 — Композитный loss для дообучения TripoSR.

### 5.4.4 Технические детали
- Распределённое обучение (если возможно): DDP / DataParallel.
- Mixed precision (fp16) для экономии VRAM.
- Gradient checkpointing для больших моделей.
-->
