## Обоснование использования Apache Kafka

<!--
Объём: 2 страницы. КРИТИЧНАЯ секция.
- Почему не синхронный gRPC между API и воркером:
  - Время инференса 30 с — соединение держать дорого.
  - Backpressure: при пике задач API падает, если воркеры заняты.
- Что даёт Kafka:
  - Buffering задач (десятки тысяч в очереди — норма).
  - At-least-once delivery (гарантия что задача не потеряется).
  - Independent scaling consumer'ов.
  - Replay для дебага (можно перепрогнать historic events).
- Альтернативы (RabbitMQ, Redis Streams) — почему Kafka:
  - Persistent log с retention.
  - Лучшая производительность на больших объёмах.

Ссылки: Kleppmann 2017, Narkhede et al. (Kafka Definitive Guide).
-->
