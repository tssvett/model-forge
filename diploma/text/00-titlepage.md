## Реферат

> Стиль заголовка: `+ЗаголРеферСодерж` ("РЕФЕРАТ" по центру).
> Стиль ключевых слов: `+КлючСловаРеферат`.
> Объём реферата: одна страница.

**Краткое содержание.** Выпускная квалификационная работа посвящена разработке микросервисной платформы для автоматической генерации 3D-моделей по 2D-изображениям с использованием методов компьютерного зрения. Проведён аналитический обзор современных подходов к однопроходной 3D-реконструкции, по результатам которого в качестве базовой модели платформы выбрана трансформерная триплан-модель TripoSR. Сформулированы функциональные и нефункциональные требования к платформе, спроектирована и реализована микросервисная архитектура, включающая REST-фасад на Kotlin/Spring Boot, ML-воркер на Python с интеграцией TripoSR, асинхронный брокер сообщений Apache Kafka, реляционное и объектное хранилища, веб-интерфейс на React и стек эксплуатационного мониторинга на основе Loki, Grafana и Prometheus. Разработана методика дообучения TripoSR на специализированных подмножествах ShapeNet и приведена количественная оценка прогнозируемого улучшения метрик качества реконструкции по литературным данным. Выполнено экспериментальное исследование производительности реализованной платформы, подтвердившее выполнение всех сформулированных нефункциональных требований.

**Объект исследования:** программные платформы для автоматизированной генерации 3D-моделей по 2D-изображениям.

**Предмет исследования:** архитектурные и алгоритмические решения, обеспечивающие выполнение требований к производительности, масштабируемости и надёжности подобной платформы.

**Цель работы:** разработка микросервисной платформы для автоматической генерации 3D-моделей по 2D-изображениям с использованием методов компьютерного зрения, а также формулировка методики дообучения базовой модели реконструкции на специализированных датасетах с прогнозом достижимого улучшения метрик качества.

**Методы:** анализ научно-технической литературы по методам 3D-реконструкции и архитектуре микросервисных систем, методы объектно-ориентированного проектирования, методы экспериментального исследования производительности программных систем – нагрузочное тестирование с применением инструмента k6 и анализ собираемых через Prometheus и Grafana метрик, а также аналитическая оценка показателей в случаях, когда прямое измерение выходит за рамки доступного аппаратного бюджета.

**Полученные результаты:** обоснован выбор TripoSR в качестве базовой модели платформы; сформулированы количественные нефункциональные требования с опорой на модель качества ISO/IEC 25010; спроектирована и реализована микросервисная архитектура с обоснованием каждого ключевого технического решения; разработана методика дообучения TripoSR на категориях ShapeNet, включающая стратегию частичной заморозки параметров и композитную функцию потерь; проведено экспериментальное исследование производительности, подтвердившее выполнение нефункциональных требований при заявленных уровнях нагрузки. Исходный код платформы, контракты API в формате OpenAPI, конфигурация развёртывания через docker-compose и сценарии нагрузочного тестирования открыты в репозитории проекта.

**Ключевые слова** (стиль `+КлючСловаРеферат`):
микросервисная архитектура, 3D-реконструкция, компьютерное зрение, TripoSR, нейронные сети, дообучение моделей, нагрузочное тестирование, Apache Kafka, FastAPI, Spring Boot.

**Объём работы:** около 80 страниц, 13 рисунков, 12 таблиц, 34 использованных источника.
