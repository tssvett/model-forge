# ЗАКЛЮЧЕНИЕ

В работе решена задача разработки микросервисной платформы для автоматической генерации 3D-моделей по 2D-изображениям с использованием методов компьютерного зрения, а также сформулирована и количественно обоснована методика дообучения базовой модели реконструкции TripoSR на специализированных подмножествах ShapeNet. Результат представляет собой целостный программный продукт с открытым исходным кодом, готовый к развёртыванию в учебном или исследовательском окружении, и набор методических положений, переносимых на родственные задачи однопроходной 3D-реконструкции.

В ходе выполнения работы решены следующие задачи. Проведён аналитический обзор современных методов 3D-реконструкции по 2D-изображениям, охватывающий классические алгоритмы фотограмметрии и метода структуры из движения, регрессионные нейросетевые модели на воксельных и точечных представлениях, нейронные излучательные поля, диффузионные подходы текст-в-3D и регрессионные трансформерные модели с триплан-репрезентацией; на основе сравнительного анализа метрик качества и вычислительных требований обоснован выбор TripoSR в качестве базовой модели платформы как наиболее благоприятного компромисса между качеством, скоростью инференса и простотой развёртывания. Сформулированы функциональные и нефункциональные требования к платформе с опорой на модель качества программных систем ISO/IEC 25010, в том числе количественные ограничения на латентность синхронных REST-операций, время инференса и пропускную способность одного ML-воркера. Спроектирована микросервисная архитектура с детальным обоснованием каждого ключевого технического решения: разделение прикладного слоя на REST-фасад на Kotlin/Spring Boot и ML-воркер на Python, выбор Apache Kafka в качестве асинхронного межсервисного брокера, паттерн transactional outbox для согласования записи в реляционную базу с публикацией события, разделение реляционных метаданных в PostgreSQL и бинарных артефактов в объектном хранилище MinIO, использование стека Loki/Grafana и Prometheus для эксплуатационного мониторинга. Реализована рабочая платформа model-forge: исходный код всех компонентов, контракты внешнего API в формате OpenAPI, конфигурация развёртывания через docker-compose и сценарии нагрузочного тестирования размещены в репозитории проекта. Разработана методика дообучения TripoSR на категориях ShapeNet с обоснованием значений гиперпараметров на литературных данных и количественной оценкой прогнозируемого улучшения метрик качества реконструкции. Проведено экспериментальное исследование производительности реализованной платформы, подтвердившее выполнение всех сформулированных нефункциональных требований при заявленных уровнях нагрузки.

**Научная новизна работы** состоит в систематической увязке функциональных и нефункциональных требований к интерактивной платформе 3D-генерации с архитектурными решениями уровня микросервисной системы, что позволяет переносить однопроходные модели реконструкции из исследовательского контекста в режим производственной эксплуатации с прогнозируемыми характеристиками качества обслуживания, и в формулировке количественно обоснованной методики дообучения регрессионной триплан-модели TripoSR на специализированных подмножествах ShapeNet, включающей стратегию частичной заморозки параметров, композитную функцию потерь и оценку прогнозируемого улучшения метрик качества реконструкции по литературным данным. Каждое архитектурное решение в работе сопровождается явным сравнением альтернатив и обоснованием выбора.

**Практическая значимость работы** определяется готовностью платформы model-forge к практическому использованию: исходный код всех компонентов открыт, конфигурация развёртывания через docker-compose позволяет поднять полноценный экземпляр на одной рабочей станции, контрактное API задокументировано в формате OpenAPI, журналирование во всех сервисах унифицировано в формате JSON со сквозным идентификатором задачи, эксплуатационный мониторинг реализован средствами стека Loki, Grafana и Prometheus. Полученные в работе количественные характеристики производительности (среднее время инференса на потребительском графическом ускорителе, эффективность горизонтального масштабирования ML-воркера, поведение системы при пиковой нагрузке) могут быть переиспользованы при оценке применимости платформы в смежных областях.

**Дальнейшие направления исследований.** Предложенная методика дообучения требует эмпирической валидации на доступной GPU-инфраструктуре с прямым сопоставлением полученных метрик с прогнозируемыми по литературным данным значениями. Перспективным направлением является сравнительное исследование альтернативных моделей реконструкции (диффузионных подходов класса Magic3D, многоракурсных регрессионных моделей класса ZeroNVS) на единой архитектурной основе платформы. На уровне инфраструктуры представляют интерес интеграция распределённой трассировки запросов через OpenTelemetry, формализация показателей качества обслуживания в виде SLI/SLO-дашбордов и подготовка платформы к развёртыванию в кластере Kubernetes для эксплуатации в режиме многоарендности. Отдельным направлением является расширение модели до приёма нескольких ракурсов входного изображения и интеграция с системой постпроцессинга мешей для подготовки результатов к использованию в средах реального времени.

Совокупность полученных результатов подтверждает достижение поставленной в работе цели: разработана полнофункциональная микросервисная платформа автоматической генерации 3D-моделей по 2D-изображениям, выполняющая все сформулированные нефункциональные требования, и подготовлена методическая база для повышения качества реконструкции на специализированных датасетах. Платформа и методика дообучения могут служить основой как для практического использования в задачах построения каталогов цифровых активов, так и для последующих научных исследований в области однопроходной 3D-реконструкции по одному изображению.
