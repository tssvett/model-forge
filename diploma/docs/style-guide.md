## 1. Что важно при написании markdown (этап автономной работы агента)

### 1.1 Язык и стиль

- **Только русский язык**, безличный научный стиль: *"в работе предложено"*, *"получено"*, *"проведён эксперимент"*. Не "я сделал", не "мы решили".
- Никакого разговорного регистра, англицизмов без необходимости. Заимствованные термины (микросервис, фреймворк, латентность) — допустимы, см. `glossary.md` для согласованного перевода.
- **Каждое утверждение** в основной части должно либо ссылаться на источник `[N]`, либо на собственную разработку (код model-forge, эксперимент в главе 7). Никаких "по нашим данным" без референса.

### 1.2 Структура работы (порядок разделов)

1. Титульный лист (без номера страницы)
2. Реферат (нумерация страниц начинается с этой)
3. Содержание
4. Перечень условных обозначений и сокращений (если нужно)
5. Введение
6. Основные главы (1, 2, 3, ... — нумерация арабскими цифрами)
7. Заключение
8. Список использованных источников
9. Приложения (Приложение А, Приложение Б, ...)

### 1.3 Иерархия заголовков (в markdown)

| Уровень | Markdown | Пример | Соответствие в финальном документе |
|---|---|---|---|
| Главные разделы (ВВЕДЕНИЕ, ЗАКЛЮЧЕНИЕ, гл. 1, гл. 2, ...) | `# 1 Название` | `# 1 АНАЛИТИЧЕСКИЙ ОБЗОР МЕТОДОВ 3D-РЕКОНСТРУКЦИИ` | ПРОПИСНЫМИ, по центру, на новой странице, без точки в конце |
| Параграф | `## 1.1 Название` | `## 1.1 Постановка задачи 3D-реконструкции` | Слева, с абзацного отступа, обычный регистр (Заголовок предложения) |
| Подпараграф | `### 1.1.1 Название` | `### 1.1.1 Классификация подходов` | Слева, обычный регистр |

**Правила нумерации:**
- Главные разделы (ВВЕДЕНИЕ/ЗАКЛЮЧЕНИЕ/СПИСОК...) — **без цифры**, только название.
- Главы — арабскими цифрами через пробел, без точки: `1`, `2`, `3`.
- Параграфы — `1.1`, `1.2`, ..., `2.1`, `2.2`, ... без точки в конце.
- Никаких точек в конце заголовков.

### 1.4 Ссылки на литературу

**Формат в тексте:** квадратные скобки с номером источника из `sources.md`.

- Одиночная ссылка: `[5]`
- Со страницей (если цитата буквальная): `[5, с. 23]` или `[5, p. 142]` для англоязычных
- Множественные: `[3, 5, 7]` или `[3—5]` (диапазон)

**Примеры:**
> Метод нейронных излучательных полей был предложен в работе [4]. Альтернативный подход через диффузионные модели описан в [7—9].

> Согласно ГОСТ 7.1-2003 [1], ...

В `sources.md` источники нумеруются в **порядке упоминания** в тексте (не по алфавиту). При написании секции — каждый новый используемый источник добавляется в `sources.md` со следующим номером, и на него идёт ссылка из текста.

### 1.5 Перечни (списки)

Из `source-materials/formatting-rules/правила_оформления_перечней.pdf`:

**Базовая форма (из СТО СГАУ):** перед каждым элементом ставится **дефис**:
```
Внутри пунктов могут быть приведены перечисления:
— первый элемент;
— второй элемент;
— третий элемент.
```

**С буквенными метками** (для последующих ссылок в тексте):
```
а) первый элемент;
б) второй элемент;
в) третий элемент.
```
(Используются строчные буквы русского алфавита, исключая: й, ё, з, о, ч, ь, ы, ъ.)

**С цифровыми метками** (для вложенных уровней):
```
1) первый;
2) второй;
3) третий.
```

**Знаки препинания в конце элементов:**
- **Запятая** — если элементы очень простые, без знаков внутри, начинаются со строчной (допустима и точка с запятой).
- **Точка с запятой** — если элементы со знаками препинания внутри, обозначены строчной буквой/цифрой со скобкой, начинаются со строчной.
- **Точка** — если обозначены номером с точкой или прописной буквой, начинаются с прописной.

**Вводное предложение** перед перечнем — обязательно. Например: *"Платформа предоставляет следующие возможности:"*. Не оставлять предлог в вводном тексте.

В markdown — записывать как обычный список, при компиляции преобразуется в нужный вид. Использовать `-` (дефис-минус) для маркеров.

### 1.6 Рисунки (иллюстрации)

**В markdown:**
```markdown
![Архитектура микросервисной системы](../assets/diagrams/architecture.png)

*Рисунок 4.1 — Архитектура микросервисной системы*
```

**Правила:**
- Подпись **под** рисунком, выравнивание по центру.
- Формат подписи: `Рисунок <глава>.<номер> — <Название>` (тире длинное `—`, не дефис).
- Нумерация **сквозная по главе**: 4.1, 4.2, 4.3.
- Ссылка в тексте — **обязательна перед рисунком**: *"Архитектура показана на рисунке 4.1"* или *"(см. рисунок 4.1)"*.
- Если рисунок взят из источника — после названия ставить ссылку: `Рисунок 2.3 — Архитектура NeRF [4]`.

### 1.7 Таблицы

**В markdown:**
```markdown
*Таблица 5.1 — Сравнение производительности микросервисов*

| Метрика | REST | gRPC | Kafka |
|---|---|---|---|
| Latency p50 | 50 мс | 15 мс | 30 мс |
```

**Правила:**
- Подпись **над** таблицей, выравнивание слева, без абзацного отступа.
- Формат: `Таблица <глава>.<номер> — <Название>`.
- Нумерация сквозная по главе.
- Если таблица переносится — на следующей странице: `Продолжение таблицы 5.1` (без названия), на последней части: `Окончание таблицы 5.1`.
- Ссылка в тексте — обязательна: *"Результаты приведены в таблице 5.1"*.

### 1.8 Формулы

**В markdown** (через LaTeX-разметку, KaTeX-совместимо):

```markdown
Расстояние Чамфера между двумя точечными облаками вычисляется по формуле:

$$
d_{CD}(P, Q) = \sum_{p \in P} \min_{q \in Q} \|p - q\|^2 + \sum_{q \in Q} \min_{p \in P} \|p - q\|^2
$$ (5.2)

где $P$ и $Q$ — точечные облака, $\|\cdot\|$ — евклидова норма.
```

**Правила:**
- Формула — на отдельной строке, по центру.
- Номер — справа в круглых скобках: `(<глава>.<номер>)`. Например: `(5.2)`.
- Ссылка в тексте: *"по формуле (5.2)"* или *"из выражения (5.2) следует, что..."*.
- Объяснение переменных — сразу после формулы, начиная со слова "где": *"где $P$ — ..."*.

### 1.9 Сокращения

- При первом упоминании — полное название, в скобках сокращение: *"глубокое обучение (deep learning, DL)"*.
- В дальнейшем — сокращение.
- Если сокращений много (>5) — отдельный раздел "Перечень условных обозначений и сокращений".

### 1.10 Перенос слов и пунктуация

- **Длинное тире `—`** (не дефис `-`) в подписях, при пояснении: *"микросервис — это..."*.
- **Дефис `-`** в составных словах: *"научно-техническая"*.
- **Кавычки** — «ёлочки» для русского текста, "лапки" допустимы для англоязычных названий.
- Между числом и единицей измерения — **неразрывный пробел**: `12 МБ`, `30 мм`, `1.25 с`.

## 2. Список использованных источников (формат ГОСТ 7.1-2003)

Извлечено из `source-materials/formatting-rules/правила_оформления_источников.pdf`.

### 2.1 Общие принципы

- Источники нумеруются **в порядке упоминания** в тексте.
- В каждом описании элементы разделяются знаком **точка-тире** (` . – `).
- Обязательные элементы выделены жирным в схемах ниже.

### 2.2 Книга (1–3 автора)

**Схема:**
```
Фамилия первого автора, инициалы. Основное заглавие [Текст] : добавочное заглавие /
инициалы и фамилии всех авторов. – Сведения о переиздании. – Место издания :
Название издательства, год издания. – Количество страниц.
```

**Пример (1 автор):**
```
Самарский, А.А. Теория разностных схем [Текст] : учеб. пособие / А.А. Самарский.
– М. : Наука, 1977. – 656 с.
```

**Пример (2 автора):**
```
Тихонов, А.Н. Уравнения математической физики [Текст] : учеб. пособие / А.Н. Тихонов,
А.А. Самарский. – М. : Наука, 1972. – 736 с.
```

**Пример (3 автора):**
```
Ильясова, Н.Ю. Информационные технологии анализа изображений в задачах медицинской
диагностики [Текст] / Н.Ю. Ильясова, А.В. Куприянов, А.Г. Храмов. – М. : Радио и связь,
2012. – 424 с.
```

### 2.3 Книга (4 и более авторов)

**Схема:** заголовок не используется, начинается сразу с основного заглавия.
```
Основное заглавие [Текст] / И.О. Фамилия первого [и др.] ; под ред. И.О. Фамилия. –
Место : Издательство, год. – Количество страниц.
```

**Пример:**
```
Методы компьютерной оптики [Текст] : учеб. для вузов / А.В. Волков [и др.] ;
под ред. В.А. Сойфера. – 2-е изд., испр. – М. : Физматлит, 2003. – 688 с.
```

### 2.4 Статья из журнала / сборника

**Схема:**
```
Фамилия, инициалы. Название статьи [Текст] / И.О. Фамилия // Название журнала. –
Место издания, год издания. – Номер тома (выпуска). – Страницы от-до.
```

**Пример:**
```
Иванов, А.А. Метод оптимизации нейронных сетей [Текст] / А.А. Иванов, Б.Б. Петров //
Известия РАН. Информационные процессы. – М., 2020. – Т. 15, № 3. – С. 45–58.
```

### 2.5 Электронный ресурс / GitHub / документация

**Схема (по ГОСТ 7.82-2001 и Р 7.0.5-2008):**
```
Фамилия, инициалы. Название [Электронный ресурс] / И.О. Фамилия. – Электрон. дан. –
Место издания : Издательство, год издания. – Режим доступа : URL (дата обращения: ДД.ММ.ГГГГ).
```

**Пример (онлайн-документация):**
```
Spring Boot Reference Documentation [Электронный ресурс]. – Электрон. дан. –
VMware Tanzu, 2024. – Режим доступа : https://docs.spring.io/spring-boot/docs/current/reference/html/
(дата обращения: 26.04.2026).
```

**Пример (статья на arXiv):**
```
Tochilkin, D. TripoSR: Fast 3D Object Reconstruction from a Single Image [Electronic resource] /
D. Tochilkin, [et al.]. – arXiv preprint arXiv:2403.02151, 2024. –
URL : https://arxiv.org/abs/2403.02151 (accessed: 26.04.2026).
```

### 2.6 ГОСТ / стандарт

```
ГОСТ 7.1-2003. Система стандартов по информации, библиотечному и издательскому делу.
Библиографическая запись. Библиографическое описание. Общие требования и правила
составления. – М. : ИПК Изд-во стандартов, 2004. – 48 с.
```

### 2.7 Базовые ГОСТы для библиографии (упоминать в списке)

- ГОСТ 7.1-2003 — Библиографическое описание
- ГОСТ 7.80-2000 — Заголовок библиографической записи
- ГОСТ 7.82-2001 — Описание электронных ресурсов
- ГОСТ Р 7.0.12-2011 — Сокращения слов на русском
- ГОСТ 7.11–2004 — Сокращения на иностранных языках
- ГОСТ Р 7.0.5-2008 — Библиографическая ссылка

## 3. Типографское оформление (для финальной компиляции в docx, этап 3)

> ⚠️ Эти параметры взяты по типичной редакции СТО СГАУ 02068410-004-2018. PDF стандарта — скан без OCR. **Проверить визуально перед финальной сдачей** или попросить актуальные параметры у научрука.

### 3.1 Поля страницы

- Левое: 30 мм
- Правое: 10 мм
- Верхнее: 20 мм
- Нижнее: 20 мм

### 3.2 Шрифт и абзац

- Шрифт: **Times New Roman**, размер **14 pt**
- Цвет: чёрный
- Межстрочный интервал: **полуторный (1.5)**
- Абзацный отступ: **1.25 см**
- Выравнивание: **по ширине**
- Перенос слов: автоматический

### 3.3 Нумерация страниц

- Сквозная, **арабскими цифрами**
- Внизу страницы, по центру (или справа — уточнить)
- На титульном листе номер не ставится, но он засчитывается (т.е. реферат — стр. 2)

### 3.4 Объём

- ВКР бакалавра: **50–70 страниц основного текста** (без титула, реферата, приложений)
- Целевой объём для нашей работы: **60–70 стр.** (8 глав по 7-9 стр. + введение + заключение)

### 3.5 Структура титульного листа

- Сверху: министерство, университет, факультет, кафедра
- По центру (1/3 страницы вниз): "ВЫПУСКНАЯ КВАЛИФИКАЦИОННАЯ РАБОТА БАКАЛАВРА"
- Тема работы: **прописными, выравнивание по центру**
- Студент / научный руководитель / зав. кафедрой — справа, с подписями
- Внизу: город, год

## 4. Чек-лист перед закрытием задачи в beads (для агента)

- [ ] Все утверждения в секции имеют ссылку на источник `[N]` или на код/собственный эксперимент.
- [ ] Иерархия заголовков соответствует пункту 1.3.
- [ ] Рисунки/таблицы/формулы пронумерованы по главе и упомянуты в тексте до появления.
- [ ] Используется длинное тире `—` в подписях и пояснениях.
- [ ] Между числом и единицей измерения — неразрывный пробел.
- [ ] Списки оформлены по правилам пункта 1.5 (дефис, или а)/б), или 1)/2)).
- [ ] Никаких русских транслитераций для уже принятых терминов из glossary.md (агент должен сверяться с глоссарием).
- [ ] При первом упоминании сокращения — полное название в скобках.
- [ ] Нет разговорных оборотов, "я", "мы решили".
